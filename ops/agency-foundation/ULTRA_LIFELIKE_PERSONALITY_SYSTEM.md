## Automation Operating Principle

The studio should automate 80-90 percent of repeatable work:

- topic selection
- hook generation
- model-specific captions and scripts
- source-image prompt generation
- image-pack sequencing
- asset indexing
- content queue creation
- approval packet generation
- telemetry receipts
- winner cloning

High-impact actions stay gated:

- HeyGen training
- public publishing
- Shopify changes
- payment or discount changes
- production deploys

## Automation Lanes

1. `identity_lock`: lock model name, alternates, age, voice, visual rules, and non-explicit boundary.
2. `image_factory`: generate 56 source images per model with continuity and safety review.
3. `heygen_factory`: create private talking avatars only after source pack and explicit approval.
4. `content_factory`: create hooks, scripts, captions, thumbnails, and social variations from the topic matrix and profile system.
5. `commerce_factory`: attach models to products, downloads, licenses, memberships, and offers only after product approval.
6. `telemetry_loop`: track performance, clone winners, and improve model/content routing.

## Immediate Production Route

1. Select priority models for full individualization.
2. Generate the 56-image source pack for each approved model.
3. Index each image with checksum, role, model ID, Drive file ID, and approval status.
4. Create HeyGen approval packet only after source pack approval.
5. Run private HeyGen tests only after explicit approval.
6. Build social/content batches from the 300-topic matrix.
7. Store receipts in Git and Drive for every external action.

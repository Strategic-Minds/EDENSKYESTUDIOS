# Eden Skye Studios Workflow Lock

## Primary Operating Loop

1. Intake
2. Classify work type
3. Rehydrate source truth
4. Confirm gates and autonomy level
5. Select avatar/persona/asset path
6. Create sandbox packet
7. Generate or draft privately
8. Validate output
9. Log telemetry and receipts
10. Request approval if required
11. Promote, publish, deploy, or mutate only after approval
12. Measure performance
13. Clone winners
14. Refill queue

## Factory Lanes

### Intake Lane

Collect request, source links, desired output, risk level, and target tool.

### Planning Lane

Convert request into packets, dependencies, gates, and validation criteria.

### Sandbox Lane

Create drafts, private media, test routes, non-public content, and schema drafts.

### Validation Lane

Check brand alignment, safety, face continuity, technical readiness, and tool receipts.

### Promotion Lane

Move approved work to production, public publishing, live scheduling, or commerce mutation.

### Improvement Lane

Review telemetry, identify winners, clone patterns, and refill queues.

## Default Daily Automation Flow

1. Read readiness and open approval state.
2. Scan trend/content inputs.
3. Generate sandbox-only draft ideas.
4. Assign avatar/persona.
5. Draft script/caption/prompt.
6. Validate against brand and safety lock.
7. Add to content queue.
8. Flag approval requests for public actions.
9. Log receipts.

## Weekly Automation Flow

1. Review content performance.
2. Identify winning topics, visuals, hooks, and CTAs.
3. Clone winners into new drafts.
4. Update missing asset list.
5. Review commerce tie-ins.
6. Create approval packet for next publishing batch.

## Failure Mode

When blocked:

1. Log blocker.
2. Name missing input/tool/access.
3. Produce workaround.
4. Keep output sandbox-only.
5. Defer live action until approval or access is restored.

# Reporting Contract

## Purpose
Standardize what the agent should return after autonomous work.

## Default report shapes
### Intelligence report
- key findings
- top opportunities
- why they matter
- best persona/platform routes
- risks or blockers

### Draft package
- recommended direction
- content drafts or hooks
- variant options
- execution notes
- approval note if needed

### Review report
- what worked
- what failed
- what changed
- what to scale
- what to stop
- what to test next

## Confidence labeling
When evidence is incomplete, explicitly label outputs as:
- verified
- inferred
- directional
- unverified

## Escalation language
If the agent stops for approval, it should say:
- what is ready
- what is blocked
- why approval is needed
- what exact next action would happen after approval

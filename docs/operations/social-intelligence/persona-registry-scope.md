# Persona Registry Scope

Status: active scope definition
Date: 2026-06-16

## Decision

The Social Intelligence Persona OS will operate with two tiers:

1. **Verified canon tier**: personas and accounts already grounded in Drive/GitHub evidence.
2. **Placeholder expansion tier**: planned platform/account rows that support routing, drafting, scoring, and approval packet preparation, but cannot be used for live execution until verified.

## Verified Canon Tier

Current verified scope:

- Eden Skye as the core brand identity.
- 12 named portrait personas from the labeled source portrait manifest.
- 5 faceless account concepts from the faceless account seed manifest.

These may be used for research, routing, scoring, drafts, and approval packets.

## Placeholder Expansion Tier

Assume the system will support all major social platforms:

- TikTok
- Instagram
- Facebook
- YouTube
- X
- Threads
- LinkedIn
- Pinterest
- Snapchat
- Reddit
- Bluesky
- Twitch
- Fanvue
- Subs.com
- OnlyFans

Placeholder rows are permitted for planning and draft routing only.

## Execution Rule

If any of these fields are placeholders, the execution ceiling is `draft_only`:

- live handle
- account ID
- page/channel/profile ID
- platform destination
- owner/approver
- publishing tool path

No placeholder account may be scheduled, posted, replied from, messaged from, paid-promoted, or connected to live avatar activation.

## Completion Criteria

The persona registry is enterprise-ready when every active persona/account pair has:

- verified persona ID
- verified platform
- verified handle
- verified account ID or page/channel/profile ID
- account owner
- approval status
- execution ceiling
- platform policy notes
- analytics source
- receipt path
- last verified date

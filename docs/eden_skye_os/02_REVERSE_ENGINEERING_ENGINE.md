# EDEN SKYE OS Reverse Engineering Engine

## Status
SCAFFOLD_ONLY / ADDITIVE SOURCE ARTIFACT.

## Purpose
The Reverse Engineering Engine studies public-facing successful content patterns, creator positioning, offer funnels, hooks, formats, thumbnails, captions, cadence, and audience response signals to create ethical, original Eden Skye Studios briefs.

## Non-Copy Rule
This engine must never copy protected creative expression, impersonate creators, scrape private data, clone identities, or reproduce proprietary assets. It extracts structural lessons only.

## Inputs
- Public trend examples
- Competitor content URLs or screenshots
- Metricool performance history
- Shopify conversion data
- Approved avatar positioning
- Offer library
- Audience research notes

## Outputs
- Pattern maps
- Hook archetypes
- Format hypotheses
- Offer-positioning insights
- Original content briefs
- Risk flags

## Required Gates
- Originality gate
- Rights/safety gate
- Brand alignment gate
- Claim verification gate
- Monetization relevance gate

## Runtime Contract
Reverse engineering is read-only analysis by default. It may generate original briefs and scoring but must not publish, schedule, spend, clone voices, use likenesses, or mutate systems without explicit approval.

## Completion Criteria
Complete when it can transform external examples into documented, original, scored content opportunities with citations, risks, and next-step routing.
